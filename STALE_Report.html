📋 Overview

STALE (State Tracking And Latent Evaluation) is a benchmark for evaluating whether LLM agents can detect and handle implicit conflicts — situations where a new observation invalidates an earlier memory without explicit negation. It contains 400 expert-validated conflict scenarios spanning everyday user-assistant dialogue with contexts up to 150K tokens. Each scenario is probed along three complementary dimensions (State Resolution, Premise Resistance, Implicit Policy Adaptation) yielding 1200 evaluation queries.
Two types of implicit conflict are tested: Type I (Co-referential) — two observations update the same underlying attribute while remaining surface-compatible; and Type II (Propagated) — a new observation updates a different attribute whose consequences cascade to invalidate an older belief through commonsense reasoning.

🔬 Three Probing Dimensions

Dim 1 · State Resolution

Can the model detect that a previously stored belief is now outdated given the new evidence? Tests basic awareness of state change.

Dim 2 · Premise Resistance

Can the model reject a query that falsely presupposes a stale state? An adversarial test that embeds outdated assumptions in the query itself.

Dim 3 · Implicit Policy Adaptation

Can the model proactively apply the updated state in downstream behavior without explicit conflict cues? Tests whether knowledge translates to action.

📊 Key Findings (from paper)

55.2% Best Accuracy

Even the best evaluated model achieves only 55.2% overall accuracy across all three dimensions — revealing a large gap between evidence retrieval and acting on it.

Premise Vulnerability

Models frequently accept outdated assumptions embedded in user queries, failing to resist adversarial premises that presuppose a stale state.

Cascade Blindness

Models struggle to recognize when a change in one attribute should invalidate structurally related memories — the harder Type II (propagated) conflict.
