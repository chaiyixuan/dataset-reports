🗂️ Dataset Schema

FieldDescription
context string - Long text context to inject into agent memory
questions list[string] - Questions to ask the agent
answers list[list[string]] - Ground truth answers (multiple acceptable answers per question)
metadata.source string - Sub-dataset source identifier
metadata.qa_pair_ids list[string] - Unique IDs for each QA pair
metadata.question_dates list[string] | null - Temporal dates for questions (LongMemEval)
metadata.question_types list[string] | null - Question type labels (LongMemEval)
metadata.haystack_sessions list | null - Session data for haystack-style tasks
metadata.keypoints list[string] | null - Key points for summarization tasks
metadata.previous_events list | null - Previous event context
metadata.demo string | null - Few-shot demonstration text
Design Philosophy: "Inject once, query multiple times" — one long context corresponds to multiple questions, significantly improving evaluation efficiency. All data is split into chunks (default 4,096 tokens) to simulate real multi-turn interaction scenarios.

🔬 Evaluation Design

🏗️ Agent Types Tested

  • Long Context Agents — GPT-4o, Claude 3.7 Sonnet, Gemini 2.0 Flash, etc.
  • Embedding RAG — text-embedding-3-small/large, Contriever
  • Structure RAG — Graph-RAG, HippoRAG, MemoRAG, RAPTOR, Cognee, Zep, Mem0
  • Agentic Memory — Letta, Self-RAG
  • Simple RAG — BM25

📏 Metrics

  • Accurate Retrieval — Exact Match / F1 Score
  • Test-Time Learning — Classification Accuracy
  • Long-Range Understanding — LLM-as-Judge (GPT-4o) / ROUGE
  • Conflict Resolution — Exact Match (on updated facts)

🆕 New Datasets

  • EventQA — Event-centric QA over long narratives (Accurate Retrieval)
  • FactConsolidation — Multi-hop facts with built-in contradictions (Conflict Resolution)
  • Detective QA — Multi-hop reasoning over mystery stories (Long-Range Understanding)
