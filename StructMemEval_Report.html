🤖 Memory Systems Evaluated

📚 mem0 RAG (top-k)

Vector database retrieval with configurable top-k. Uses Qdrant for embedding storage and OpenAI embeddings. Tests how well flat vector search handles structured memory.

🔧 mem0 Agent

Tool-calling agent with add/search/update/delete memory operations. The agent actively manages its own memory through function calls. Configurable iterations and search limits.

📝 mem-agent (Structured)

Structured markdown file-based memory. Organizes information in user.md profile files and per-entity markdown documents. Tests if structured memory outperforms flat retrieval.

💡 Key Insights

🎯 Core Research Question

Does structured memory (entity-based markdown) outperform flat vector retrieval (RAG) for LLM agents? StructMemEval systematically tests this across diverse task types.

📐 Scaling Dimensions

Each category tests different scaling axes: accounting varies entry count (10/30/50), recommendations vary item count (100-350), tree-based varies graph depth, state machine varies transition count.

🔍 Noise Injection

Datasets include distractor messages (e.g., "The music makes this trip special!" in accounting) to test whether memory systems can filter signal from noise.

⚖️ Multi-Answer Tolerance

Many queries accept multiple reference answers (e.g., different valid debt settlement plans), testing for semantic equivalence rather than exact string matching.
